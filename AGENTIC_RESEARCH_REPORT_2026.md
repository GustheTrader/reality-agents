## The 7 Agent Protocols You Need to Know

### 1. MCP (Model Context Protocol) — Anthropic
**The dominant agent-to-tool layer.**

- Created by Anthropic
- Adopted by OpenAI, Google, Microsoft
- Standardizes how agents connect to tools (databases, APIs, files)
- Think of it as "USB for AI agents"
- Won the agent-to-tool layer

**Status:** Effectively the standard. 146 members in the Agentic AI Foundation.

### 2. A2A (Agent-to-Agent Protocol) — Google
**Standardizes agent-to-agent communication.**

- Created by Google
- Focuses on how agents talk to each other (not tools)
- Complementary to MCP (MCP = agent↔tool, A2A = agent↔agent)
- Contributed to the Agentic AI Foundation

**Status:** Growing adoption. Complements MCP.

### 3. AGENTS.md — OpenAI
**Standardizes agent configuration and behavior.**

- Created by OpenAI
- File-based agent configuration
- Similar to how README.md works for projects
- Contributed to the Agentic AI Foundation

**Status:** Early adoption.

### 4. Goose — Block (Square)
**Open-source AI agent.**

- Created by Block (Square)
- Open-source, extensible
- Contributed to the Agentic AI Foundation

**Status:** Growing community.

### 5. ACP (Agent Communication Protocol) — IBM
**Enterprise agent communication.**

- Created by IBM
- Focus on enterprise-grade agent communication
- Security, compliance, auditability

**Status:** Enterprise focus.

### 6. UCP (Universal Connection Protocol)
**Universal agent connection standard.**

- Cross-platform agent connections
- Vendor-neutral

**Status:** Emerging.

### 7. Our ATA Protocol (Sovereign OS)
**Encrypted, sovereign, peer-to-peer.**

- Created by us
- Encrypted by default
- Sovereign (no corporate middleman)
- Mesh network architecture
- Supports any LLM backend

**Status:** Built. Ready to integrate with MCP/A2A.
